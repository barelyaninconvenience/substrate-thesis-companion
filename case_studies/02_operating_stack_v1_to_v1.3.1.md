**Status:** Tight seed 2026-04-24. Companion to case_studies/01.

## Why this case complements case 01

Case 01 (Descriptive Insights v1 → v2) shows the reinvigoration template applied to a **content artifact** with a discrete audit catalyst.

Case 02 (Operating Stack v1 → v1.1 → v1.2 → v1.3.1) shows the same template applied to a **framework artifact** that evolved continuously through multiple sub-versions, each catalyzed by lived experience rather than a discrete audit event.

Together they bracket the template's range: artifact-with-audit on one end, framework-with-experience on the other.

- L1 (model) → L2 (effort) → L3 (verbosity) → ... → L14 (gate-and-posture multiplier)

### SRD instance

## What this case adds beyond case 01

1. **Continuous-evolution catalyst** vs case 01's discrete-audit catalyst — both work; the template is catalyst-agnostic
2. **Sub-version granularity** (v1.0 → v1.1 → v1.2 → v1.3.1) vs case 01's binary v1 → v2 — template handles both
3. **Quantitative calibration** (within 5% of empirical) vs case 01's narrative-quality refinements — template applies regardless of artifact type
4. **Faster cycle time** (4 days vs 3 weeks) — calibration: framework artifacts iterate faster because lived experience is continuous; content artifacts iterate at audit cadence
